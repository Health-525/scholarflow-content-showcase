# 运动记录输入样例

## 场景

用户不需要维护复杂表格，只通过最简单的复选框记录完成情况，系统再负责统计与回写。

## 用户输入

### 月度记录视图

```md
## 2026-03

- [x] 03-08
- [ ] 03-10
- [x] 03-12
- [x] 03-15
```

### 更密集的记录示例

```md
## 2026-03

- [x] 03-01
- [x] 03-03
- [ ] 03-04
- [x] 03-06
- [ ] 03-09
- [x] 03-10
- [x] 03-12
- [x] 03-15
- [ ] 03-17
- [x] 03-19
- [x] 03-21
- [ ] 03-24
- [x] 03-27
```

## 为什么这种输入方式成立

- 勾选框天然适合移动端和碎片化记录
- 用户不需要手动算频率、趋势或完成率
- 与 Markdown 笔记风格兼容，不破坏内容层一致性

## 系统会做什么

执行层可以把这些离散记录转换为结构化统计，用于：

- 计算完成次数与间隔
- 生成趋势汇总
- 在日报、周报中引用运动行为摘要
- 进一步生成可视化热力图或简单统计指标

## 最终结果会出现在哪里

- 执行层中的运动记录样例状态
- 内容层中的日报、周报或统计型可视化结果

## 公开展示重点

这个样例说明：

> 内容层只负责记录“发生了没有”，而统计、汇总和解释都可以交给执行层。

## 脱敏说明

展示版只保留最基础的输入形态，不保留真实日期节奏或与个人生活规律直接相关的长期记录。
