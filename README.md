# jiangshu-study-showcase

> ScholarFlow 内容层展示骨架仓库
>
> 用于向评委或访客展示 Obsidian 内容层如何作为多 Agent 学习系统的统一输入与阅读界面。

---

## 仓库定位

这个仓库不是作者的真实私人知识库，而是一个 **公开展示版骨架**。

它保留了内容层的核心结构、交互入口和产物形态，但做了以下处理：

- 移除真实姓名、学号、班级、学校等身份信息
- 移除真实课表、地点、时间轨迹和连续行为记录
- 只保留少量脱敏样例，用于说明系统如何输入、处理和回写

---

## 你能在这里看到什么

- Obsidian 风格的课程与知识目录组织
- `09-日常处理/` 作为系统交互入口的设计方式
- 自动生成产物在内容层的呈现样式
- 日报、周报、视频笔记、知识研究等样例输出

---

## 核心结构

```text
jiangshu-study-showcase/
├── 00-项目说明/          # 系统说明与使用方式
├── 01-课程样例/          # 脱敏后的课程笔记/作业样例
├── 08-AI学习/            # 知识研究类样例
├── 09-日常处理/          # 内容层交互入口
├── 日报/                 # 日报样例
├── 周报/                 # 周报样例
├── youtube-daily/        # 视频笔记样例
├── _meta/                # 非个人化的系统角色说明
└── 图片/                 # 展示截图或配图
```

---

## 内容层在系统中的作用

在 ScholarFlow 中，内容层只负责两件事：

1. 作为用户的轻量输入入口
2. 作为系统产物的可读展示面

用户继续在熟悉的 Markdown / Obsidian 环境中工作，不需要切换到新的产品界面。执行逻辑、状态存储和自动化调度则放在独立的执行层仓库中。

---

## 与执行层的关系

- 内容层：本仓库，负责阅读、编辑、沉淀
- 执行层：`timetable-showcase`，负责脚本、工作流、数据处理和自动回写

建议结合执行层展示仓库一起看，能够更完整理解双仓库协同架构。
