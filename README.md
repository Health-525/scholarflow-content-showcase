# scholarflow-content-showcase

> ScholarFlow 的内容层公开展示仓库，展示用户如何用最少量的 Markdown 输入，驱动一套持续运行的学习型多 Agent 系统。

[执行层仓库](https://github.com/Health-525/scholarflow-engine-showcase) · [公开展示边界](PRIVACY.md) · [双仓库协作说明](00-项目说明/双仓库协作说明.md)

---

## 30 秒看懂

ScholarFlow 不把学习管理做成一个全新的重产品，而是保留用户已经熟悉的 Obsidian / Markdown 工作方式：

1. 用户继续在内容层仓库里做轻量记录
2. 执行层自动解析输入、更新结构化状态、生成产物
3. 课表、日报、周报、研究笔记再回写到内容层

```mermaid
flowchart LR
    A[用户在 Obsidian 中追加少量记录] --> B[内容层仓库保存 Markdown]
    B --> C[执行层解析输入并更新样例状态]
    C --> D[系统生成课表 / 日报 / 周报 / 研究产物]
    D --> E[产物回写到内容层继续阅读与复盘]
```

这个仓库的重点不是展示“所有自动化逻辑”，而是展示内容层为什么能成为稳定的人机协作界面。

---

## 这个仓库解决什么问题

普通学习记录工具往往只负责“记下来”，但不会持续接手后续处理。ScholarFlow 的内容层展示了另一种思路：

- 输入尽量轻：继续使用 Markdown，而不是要求用户迁移到新系统
- 反馈尽量快：执行层收到输入后可以生成新的提醒、总结和产物
- 沉淀尽量长：课表、日报、周报和研究笔记最终都回到同一个知识库里

## 公开展示边界

这个仓库不是原始私人知识库，而是脱敏后的公开展示版。

- 不包含真实姓名、学号、班级、学校信息
- 不包含真实课表、真实地点、连续时间轨迹
- 不包含完整历史日志、自动同步凭据或私有上下文
- 保留的是交互方式、目录组织和产物形态

更详细的边界说明见 [PRIVACY.md](PRIVACY.md)。
